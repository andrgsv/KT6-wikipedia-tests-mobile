# Тестовый план

## 1. Объект тестирования

Мобильное приложение: Wikipedia для Android.

Тип приложения: информационное мобильное приложение для поиска и чтения статей.

Платформа: Android.

## 2. Цель тестирования

Проверить основные пользовательские функции мобильного приложения Wikipedia и оформить тестовую документацию по результатам проверки.

## 3. Задачи тестирования

- проверить запуск приложения;
- проверить прохождение первого экрана / онбординга;
- проверить поиск статьи;
- проверить открытие статьи из результатов поиска;
- проверить сохранение статьи;
- проверить переход в настройки;
- зафиксировать найденные дефекты;
- подготовить итоговый отчет.

## 4. Типы тестирования

| Тип тестирования | Описание |
|---|---|
| Функциональное | Проверка основных функций приложения |
| Smoke | Проверка базовой работоспособности приложения |
| Регрессионное | Проверка, что базовые сценарии не сломаны |
| UI-тестирование | Проверка доступности основных элементов интерфейса |
| Usability | Проверка удобства навигации и понятности интерфейса |

## 5. Объем тестирования

Входит в тестирование:

- запуск приложения;
- онбординг;
- поиск;
- открытие статьи;
- сохранение статьи;
- навигация в настройки.

Не входит в тестирование:

- регистрация нового пользователя;
- авторизация;
- редактирование статей;
- проверка серверной части Wikipedia;
- нагрузочное тестирование;
- безопасность аккаунта.

## 6. Среда тестирования

| Параметр | Значение |
|---|---|
| ОС компьютера | Windows 10/11 или Linux |
| Язык программирования | Python |
| Инструмент автоматизации | Appium |
| Тестовый фреймворк | Pytest |
| Платформа | Android |
| Устройство | Эмулятор Android или реальный Android-телефон |
| Приложение | Wikipedia |
| App package | org.wikipedia |

## 7. Критерии начала тестирования

- установлены Python, Appium, Android SDK;
- запущен Appium Server;
- доступно Android-устройство или эмулятор;
- установлено приложение Wikipedia;
- устройство отображается в `adb devices`;
- подготовлены тест-кейсы.

## 8. Критерии завершения тестирования

- выполнены все запланированные тест-кейсы;
- зафиксированы результаты прохождения;
- найденные дефекты описаны в отчетах;
- подготовлен итоговый отчет;
- даны рекомендации по улучшению.

## 9. Риски

| Риск | Влияние | Способ снижения |
|---|---|---|
| Изменение интерфейса после обновления приложения | Тесты могут упасть | Использовать Appium Inspector для обновления локаторов |
| Разные языки интерфейса | Тесты могут не найти текст | Использовать resource-id и fallback-локаторы |
| Нестабильное интернет-соединение | Поиск может работать медленно | Использовать явные ожидания |
| Медленный эмулятор | Увеличение времени выполнения тестов | Использовать реальное устройство или повысить timeout |
