# Специфика платформы Android для тестирования

## Особенности Android, влияющие на тестирование

1. Разные версии Android.
   Приложение может вести себя по-разному на Android 10, 11, 12, 13, 14 и 15.

2. Разные размеры экранов.
   Элементы интерфейса могут отображаться иначе на телефонах и планшетах.

3. Разные языки интерфейса.
   Текстовые локаторы могут перестать работать, если язык устройства изменен.

4. Разрешения приложения.
   Некоторые функции могут зависеть от разрешений Android.

5. Состояние сети.
   Мобильное приложение должно корректно работать при медленном интернете или отсутствии сети.

6. Жизненный цикл приложения.
   Нужно учитывать сворачивание, возвращение из фона и перезапуск приложения.

## Ограничения тестирования

- Не выполнялось нагрузочное тестирование.
- Не тестировалась авторизация пользователя.
- Не проверялось редактирование статей.
- Не проводилось тестирование безопасности.

## Рекомендации

- Использовать реальные устройства и эмуляторы.
- Применять явные ожидания вместо `time.sleep()`.
- Использовать `resource-id` и `accessibility-id`.
- Проверять приложение при разных настройках языка и сети.
