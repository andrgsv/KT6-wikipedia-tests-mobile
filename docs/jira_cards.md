# Примеры карточек Jira / YouGile / Kaiten

## Карточка BUG-001

**Тип задачи:** Bug  
**Summary:** Поиск не показывает понятное сообщение при отсутствии интернета  
**Priority:** Medium  
**Severity:** Medium  
**Labels:** mobile, android, search, usability  
**Assignee:** Android developer  

**Description:**

При отключенном интернет-соединении пользователь не получает достаточно понятного сообщения о проблеме. Поиск может находиться в состоянии загрузки.

**Steps to reproduce:**

1. Отключить интернет на Android-устройстве.
2. Запустить Wikipedia.
3. Открыть поиск.
4. Ввести `Selenium`.

**Expected result:**

Показывается понятное сообщение об отсутствии подключения.

**Actual result:**

Результаты не отображаются, состояние ошибки не всегда очевидно.

**Recommendation:**

Добавить сообщение `Нет подключения к интернету` и кнопку `Повторить`.

## Карточка BUG-002

**Тип задачи:** Bug  
**Summary:** Тесты навигации зависят от языка интерфейса  
**Priority:** Medium  
**Severity:** Low  
**Labels:** autotest, appium, android, localization  
**Assignee:** QA automation engineer  

**Description:**

Автотесты могут быть нестабильными, если элементы ищутся только по тексту. При смене языка устройства текст кнопок изменяется.

**Steps to reproduce:**

1. Запустить тест перехода в настройки на устройстве с английским языком.
2. Запустить тот же тест на устройстве с русским языком.
3. Сравнить результат.

**Expected result:**

Тесты работают стабильно на разных языках.

**Actual result:**

Тест может упасть при поиске элемента по тексту.

**Recommendation:**

Использовать стабильные локаторы: `resource-id`, `accessibility-id`, fallback-локаторы.
