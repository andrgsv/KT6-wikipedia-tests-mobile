# Отчеты о дефектах

## BUG-001. Неочевидное поведение поиска при отсутствии интернет-соединения

| Поле | Значение |
|---|---|
| ID | BUG-001 |
| Модуль | Поиск |
| Серьезность | Средняя |
| Приоритет | Средний |
| Статус | Open |
| Окружение | Android, Wikipedia app, Appium/Pytest |

Описание:

При отключенном интернет-соединении поиск статьи может долго оставаться в состоянии загрузки. Пользователю не всегда сразу понятно, что проблема связана с отсутствием сети.

Шаги для воспроизведения:

1. Отключить интернет на устройстве.
2. Открыть приложение Wikipedia.
3. Нажать на строку поиска.
4. Ввести запрос `Selenium`.
5. Дождаться результата.

Ожидаемый результат:

Приложение быстро показывает понятное сообщение, например: `Нет подключения к интернету`.

Фактический результат:

Результаты поиска не отображаются, пользователь видит длительную загрузку или неочевидное состояние.

Рекомендация:

Добавить явное сообщение об отсутствии интернет-соединения и кнопку повторной попытки.

## BUG-002. Возможная нестабильность автотестов из-за изменения языка интерфейса

| Поле | Значение |
|---|---|
| ID | BUG-002 |
| Модуль | UI / Навигация |
| Серьезность | Низкая |
| Приоритет | Средний |
| Статус | Open |
| Окружение | Android, разные языковые настройки устройства |

Описание:

При разных языках интерфейса названия кнопок и пунктов меню могут отличаться. Это влияет на стабильность автотестов, если используются текстовые локаторы.

Шаги для воспроизведения:

1. Установить язык устройства на русский.
2. Запустить автотест, который ищет кнопку по английскому тексту `Settings`.
3. Повторить тест на устройстве с английским языком.

Ожидаемый результат:

Тесты должны стабильно работать независимо от языка интерфейса.

Фактический результат:

Тест может не найти элемент, если локатор зависит только от текста.

Рекомендация:

Использовать `resource-id`, `accessibility-id` и fallback-локаторы.
