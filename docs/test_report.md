## 2. Краткое описание

В рамках работы было протестировано мобильное приложение Wikipedia. Проверялись основные функции: запуск приложения, пропуск онбординга, поиск статьи, открытие статьи, сохранение статьи и переход в настройки.

|---|---:|
| Passed | 6 |
| Failed | 0 |
| Blocked | 0 |

## 4. Найденные дефекты

| ID | Название | Серьезность | Приоритет | Статус |
|---|---|---|---|---|
| BUG-001 | Неочевидное поведение поиска при отсутствии интернет-соединения | Средняя | Средний | Open |
| BUG-002 | Возможная нестабильность автотестов из-за изменения языка интерфейса | Низкая | Средний | Open |

## 5. Анализ качества приложения

Приложение Wikipedia выполняет основные функции корректно. Главные пользовательские сценарии доступны без регистрации: пользователь может открыть приложение, найти статью, перейти к чтению и сохранить материал.

Основной риск связан не с критическими ошибками приложения, а с условиями мобильной среды: отсутствием интернета, изменением языка интерфейса, разными версиями Android и возможными изменениями локаторов после обновления приложения.

## 6. Рекомендации

- Добавить более заметное сообщение при отсутствии интернет-соединения.
- При автоматизации использовать стабильные локаторы, а не только текст элементов.
- Проверять приложение минимум на одном эмуляторе и одном реальном устройстве.
- Добавить тесты на смену ориентации экрана.
- Добавить тесты на поведение приложения при медленном интернете.

## 7. Вывод

Цель работы достигнута. Было выбрано реальное мобильное Android-приложение, составлен тестовый план, разработаны тест-кейсы, подготовлены автотесты Appium/Pytest, оформлены отчеты о дефектах и итоговый отчет по результатам тестирования.
